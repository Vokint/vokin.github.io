Apply structural programming approach to solve more complex computation problems.
Demonstrate applications of standard data structures such as linked list, stack, queue and tree.
Solve computation problems using recursion where appropriate.

Chapter 2 - Computation Complexity (Big-O)

There are two types of complexity

  • Time Complexity (Time spent)
  • Space Complexity (Memory space spent)

They both use three asymptotic notation to descript the complexity

  • Big-O (O)\textnormal{Big-O (O)}Big-O (O)
  • Big-Omega (Ω)\textnormal{Big-Omega }(\Omega)Big-Omega (Ω)
  • Big-Theta (Θ)\textnormal{Big-Theta }(\Theta)Big-Theta (Θ)

Big-O

Upper bound of the complexity

f(n)=O(g(n))f(n) = \textnormal{O}(g(n)) f(n)=O(g(n))

Big-Omega

Lower bound of the complexity

f(n)=Ω(g(n))f(n) = \Omega(g(n)) f(n)=Ω(g(n))

Big-Theta

Exact bound of the complexity

f(n)=Θ(g(n))f(n) = \Theta(g(n)) f(n)=Θ(g(n))

Common Complexity Classes

  • Constant time - O(1)O(1)O(1)
  • Logarithmic time - O(log⁡2n)O(\log_2 n)O(log2​n)
  • Linear time - O(n)O(n)O(n)
  • Log-linear time - O(nlog⁡2n)O(n\log_2 n)O(nlog2​n)
  • Quadratic time - O(n2)O(n^2)O(n2)
  • Cubic time - O(n3)O(n^3)O(n3)
  • Polynomial time - O(nk)O(n^k)O(nk)
  • Exponential time - O(2n)O(2^n)O(2n)
  • Factorial time - O(n!)O(n!)O(n!)

Slope of Complexity Classes

TIme Complexity

Simple operations such as

  • declaration, assignment
  • cin, cout
  • function call
  • return

take constant amount of time
The overall time complexity = O(Constant),
and constant time costs almost zero influence no matter the size.
Therefore the overall time complexity is

O(1)O(1) O(1)

Only looping will cost longer time complexity
Single nested loop usually takes constant time * size of loop as the operations C inside the loop will run n times.

C∗nC * n C∗n

The overall time complexity is

O(n)O(n) O(n)

While for more than one nested loop usually takes constant time * (n ^ depth of nest) as the operations C inside the loop will run n * n * ... times.

C∗n2C * n^2 C∗n2

The overall time complexity is

O(n2)O(n^2) O(n2)

Josephus Problem

People are standing in a circle waiting to be executed.
Counting begins at a specified point in the circle and
proceeds around the circle in a specified direction. After
a specified number of people are skipped, the next
person is executed. The procedure is repeated with the
remaining people, starting with the next person, going in
the same direction and skipping the same number of
people, until only one person remains, and is freed

The problem — given the number of people (n), starting
point, direction, and number to be skipped (k) — is to
choose the position in the initial circle to avoid execution.
(guessing who is the survivor)

Introduction of Hashing

Hashing is generally [ Indexing ].

If you have learnt what [ index ] is, you may understand it as positioning or classifying objects with duplicated meaning for a smaller size of data and better searching.

Steps for Indexing

  1. Define an array with a specific purpose.
  2. Define meaning for each index in the array.
  3. Classify the input and determine the size of the array.
  4. Map the input to the array.

Example of Indexing (Minimal Perfect Hashing)

Given A string [foo] = “abacbdef”.
Precondition: there will only be alphabets inside the string.
If you want to find how many ‘a’ occured inside the string, you may use Indexing.

  1. Define an array for storing the occurrence of letters.
  2. Each index represents an alphebet.
  3. There are a total of 26 alphabets as potential input, so the size of array will be 26.
  4. Similar to the ASCII table, we can take index [0] representing the occurrence of ‘a’, index [1] representing the occurrence of ‘b’, and so on…
  5. Loop over the string and map the character to corresponding index.

However, there are still some differences between Hashing and Indexing.
In Hashing, we convert ASCII char to key directly with array[i] - 'a' so that ‘a’ = 0, ‘b’ = 1, and so on…
which allows us to map the data easier and faster.

Hash function

A function f(k) is known as defined with specific formula,
it can be either One-to-one, or Many-to-one,
Under this property,
it can convert a large variable-sized amount of data into a small datum by transforming data into keys.

A Hash table arr[ ] is an array that will be created to hold the keys after conversion.

A Hash value is usually an integer that may serve as an index to an array.

Example of Hash function

Hashing Student ID k using Modulo, where h is the function, h(k) is the hash value of k, n is an arbitrary key.

h(k)=k mod nh(k) = k \bmod n h(k)=kmodn

The size of the array will be reduced significantly due to the limited range of the modulus [0 to n-1].

Take k = 5000, n = 5, the function will be

h(k)=k mod 5h(k) = k \bmod 5 h(k)=kmod5

In Hash table, the value k will be updated to the index of hash value, arr[h(k)] = k

h(5000)=5000 mod 5=0h(5000) = 5000 \bmod 5 = 0 h(5000)=5000mod5=0

arr[0] = 5000, arr[1] = 5081, arr[2] = 5072, arr[3] = 5023, arr[4] = 5014

You may now be thinking of what if the index collides, it should happen frequently as the function is many-to-one.

Yes, and therefore, I am going to introduce the solution against collision.

Hash Collision

Collision occurs when two distinct data own the same hash value.
Collisions are usually unavoidable when a large set of data is mapped into small datum.
It will increase the average cost of lookup operations.

Solutions to Hash Collision

  • Open Addressing
  • Closed Addressing

Load factor represents the length of data in each index,
larger load factor brings lower performance

Open Addressing

Place the key in other free slot when collisions occured.

An index only maps to 1 key.
Theoretical max load factor = 1

Advantages

  • No size overhead apart from the hash table array
  • Better memory locality and cache performance. Memory allocation is laid out linearly
  • Performs better when the number of keys is known in advance and the churn is low.

After dealing with collision, there will be two general types of Hashing.

Minimal Perfect Hashing (Best)

A hash function that maps each valid input to a different hash value.
Every index is taken in the hash table.

Perfect Hashing

A hash funciton that maps n input keys to n consecutive hash values.
But not every index is taken in the hash table.

Designing Hash Functions

Ideal properties

  • Low Cost
    • Easier and faster to compute the result
  • Variable Range
    • Able to transform [words], [symbols] into [numbers]
  • Uniformity
    • Distribute the keys evenly
    • Minimize the chance of collisions

Common types of Hash Functions

  1. Division

  2. Mid-Square

  3. Folding

  4. Radix Transformation

Division

hashValue = key % tableSize
Try to use prime number as the hash table size to reduce collisions

For non integer keys, transform them into integer with ASCII code

Mid-Square

  1. Transform keys into integer
  2. Sqaure the number
  3. Pick some digits from the middle to hash

Take tableSize = 5

key\textnormal{key}key key2\textnormal{key}^2key2 new key\textnormal{new key}new key hashValue\textnormal{hashValue}hashValue
99 9801 80 80 % 5 = 0
145 21025 102 102 % 5 = 2
342 116964 69 69 % 5 = 4

Folding

  1. Split key into 3 parts [ Left, Middle, Right ] as you wish,
    left and right should have the same size
  2. Fold keys and sum the keys

Shift-folding

Sum the splitted keys directly

Take tableSize = 5

key\textnormal{key}key split\textnormal{split}split new key\textnormal{new key}new key hashValue\textnormal{hashValue}hashValue

1234567 123-4-567 123 + 4 + 567 = 694 694 % 5 = 4

Boundary-folding

Invert the left and right part of each splitted part except middle one
Then sum the keys
